# The Story Behind Mosque Finder

Every Muslim is required to perform the five daily prayers on time.

Finding a nearby mosque is usually easy.

Finding its actual congregation (Jamaat) timing is not.

This became especially noticeable after visiting a metropolitan city where mosque schedules differed significantly from what I was familiar with. Some mosques remained open throughout the day, while others opened only shortly before prayer. Congregation timings varied from one mosque to another, making it difficult to know whether there was still enough time to reach the mosque before the prayer began.

Although many applications provide calculated prayer times, very few provide the information people actually need before leaving for the mosque.

Questions such as:

- Is the mosque currently open?
- When does the congregation begin?
- Can I still reach it in time?
- Does it have Wudu facilities?
- Is there a prayer area for women?

often remain unanswered.

Mosque Finder was created to solve that problem.

Its purpose is simple:

> **Help people spend less time searching and more time praying.**

Mosque Finder exists to make reliable mosque information easily accessible so that Muslims can confidently find a nearby mosque, know its congregation timing, and perform Salah on time wherever they are.

# Engineering Highlights

Mosque Finder has been designed with long-term maintainability, reliability, and scalability in mind.

---

## 📅 Smart Timetable Importer

Mosques and cities often publish yearly prayer calendars in different spreadsheet formats.

To simplify administration, Mosque Finder includes a deterministic timetable importer capable of validating and importing calendars without requiring manual data entry.

### Supported Features

- CSV Import
- XLS Import
- XLSX Import
- Automatic Header Recognition
- Prayer Name Aliases
- Multi-Sheet Workbook Detection
- Import Preview
- Merge Mode
- Replace Mode
- Validation Reports
- Leap-Year Awareness
- Transaction-Safe Imports

The importer validates every file before writing anything to the database, allowing administrators to confidently review the results before confirming an import.

---

## ⚡ Live Mosque Availability Engine

Instead of displaying only today's prayer timings, Mosque Finder determines whether a mosque is currently open.

The availability engine evaluates:

- Current local time
- Daily prayer schedule
- Congregation timings
- Opening window
- Closing window

Users immediately know whether travelling to a mosque is worthwhile before leaving.

---

## 📍 Intelligent Location Processing

Mosque administrators can register a mosque simply by providing its Google Maps location.

The backend automatically extracts and validates geographic coordinates, stores normalized location data, and prepares it for map rendering and nearby search.

This significantly reduces manual data entry while improving location accuracy.

# Engineering Decisions

Several architectural decisions shape the way Mosque Finder works.

These decisions prioritize correctness, maintainability, and simplicity over unnecessary complexity.

---

## Community-Managed Information

Mosque Finder does not attempt to estimate congregation timings.

Instead, it provides congregation information maintained by mosque administrators, making the platform more useful for people planning to attend prayers.

---

## Deterministic Timetable Import

Prayer calendars are validated before any data is written to the database.

Administrators can preview:

- Errors
- Warnings
- Duplicate entries
- Import summary

before confirming the import.

This minimizes the risk of incorrect prayer schedules.

---

## Separation of Responsibilities

Different modules manage different domains.

For example:

- Cities manage city information.
- Mosques manage mosque-specific information.
- Prayer services manage prayer-related logic.
- Platform Administration manages operational workflows.

Keeping these responsibilities separate improves maintainability and reduces unintended side effects.

---

## Single Source of Truth

Mosque Finder avoids storing duplicated information wherever possible.

Examples include:

- Daily prayer schedules are stored once per city.
- Mosque records reference city information.
- Administrative workflows are separated from operational data.

Maintaining a single source of truth improves consistency and simplifies long-term maintenance.

# Backend Setup

Navigate to the backend directory.

```bash
cd backend
```

Create a virtual environment.

```bash
python -m venv venv
```

Activate the virtual environment.

### Windows

```bash
venv\Scripts\activate
```

### Linux / macOS

```bash
source venv/bin/activate
```

Install project dependencies.

```bash
pip install -r requirements/local.txt
```

Create your environment file.

```bash
cp .env.example .env
```

Apply database migrations.

```bash
python manage.py migrate
```

Create a superuser (optional).

```bash
python manage.py createsuperuser
```

Run the backend server.

```bash
python manage.py runserver
```

The backend will be available at:

```
http://127.0.0.1:8000
```

---

# Frontend Setup

Open a new terminal.

Navigate to the frontend.

```bash
cd frontend
```

Install dependencies.

```bash
npm install
```

Create the frontend environment file.

```bash
cp .env.example .env.local
```

Run the development server.

```bash
npm run dev
```

The frontend will be available at:

```
http://localhost:3000
```
